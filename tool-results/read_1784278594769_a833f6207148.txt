   270→Root Cause Analysis:
   271→====================
   272→1. NO PROCESS MANAGER: The dev server and mini-service were started as orphaned
   273→   background processes (`bun run dev &` + `disown`). When they crashed or the
   274→   container restarted, NOTHING restarted them. The mini-service was frequently
   275→   dead (port 3002 not listening).
   276→
   277→2. DESTRUCTIVE STARTUP SCRIPT: `.zscripts/dev.sh` ran `bun run db:push` on EVERY
   278→   container startup. While Prisma db:push is non-destructive for schema changes,
   279→   it can potentially reset data in certain scenarios, causing session loss.
   280→
   281→3. SHORT SESSION EXPIRY: NextAuth JWT expired after 7 days. Combined with
   282→   container restarts, users were frequently forced to re-login.
   283→
   284→4. NO GIT PERSISTENCE: Previous session's work (Macro engine, Swing tab) was
   285→   written to disk but lost during container restart because it wasn't committed
   286→   before the restart happened. The automated snapshot system only captures
   287→   what's on disk at snapshot time.
   288→
   289→5. AUTOMATED GIT COMMITS: An external system periodically commits the working
   290→   tree with UUID messages (e.g., "641adc1e-db69-4f5b-bf1f-5fe58402cb9a").
   291→   These commits are snapshots, not human commits.
   292→

   409→ROOT CAUSE (FINAL ANSWER):
   410→==========================
   411→The login page persisted because NextAuth's CSRF protection SILENTLY REJECTED
   412→all login attempts that came through the Caddy gateway (port 81). The gateway
   413→forwards the browser's Host header (e.g., "localhost:81" or the preview panel
   414→URL), but NextAuth was comparing it against the internal server URL
   415→(localhost:3000). When they didn't match, NextAuth:
   416→1. Redirected to /api/auth/signin?csrf=true (CSRF failure)
   417→2. Did NOT set the next-auth.session-token cookie
   418→3. The browser stayed on the login page indefinitely
   419→
   420→This was NOT a design revert, NOT a code loss, NOT a session expiry issue.
   421→It was a CSRF validation failure caused by missing `trustHost: true` in the
   422→NextAuth configuration.
   423→
   424→THE FIX:
   425→========
   426→Added to src/lib/auth.ts:
   427→  trustHost: true,
